优雅的使用async和await

2019-11-11 / 3 min read

今天是光棍节，起名字得起个不太一样的(莫名感觉自己有点大佬的味儿了)

😏表明看起来高大上，其实就是简单介绍一下两个语法的使用

一、优雅的介绍一下async和await

async 函数是什么？一句话，它就是 Generator 函数的语法糖。
async函数就是将 Generator 函数的星号（*）替换成async，将yield替换成await，仅此而已。

                                                                    ————阮一峰

阮大佬的入门教程如是说道，那么我们就大概知道了async/await也是用来实现异步编程的方式。

二、上demo，冲！

在 function 前面加 async 用来说明这个函数是一个异步函数
不多说代码安排上：

      let sayHi = async () => {
        console.log('hello,飞飞鱼');
      }
      sayHi();
      console.log('hello,丁丁车');

看下控制台的打印：

emm？👀这不还是同步吗？
再去看看，async 函数返回的是一个promise 对象，喔，懂了！

      let sayHi = async () => {
        return 'hello,飞飞鱼';
      }
      sayHi()
      console.log('hello,丁丁车');

打开命运石之门：


这一切都是命运石之门的选择。

好吧，为什么没有打印出来呢？我们把函数sayHi执行结果打印一下：

      let sayHi = async () => {
        return 'hello,飞飞鱼';
      }
      console.log(sayHi());
      console.log('hello,丁丁车');


额， 忘了返回的是promise对象，那我们直接.then()取值。

      let sayHi = async () => {
       return 'hello,飞飞鱼';
      }
      sayHi().then(res => console.log(res));
      console.log('hello,丁丁车');


秒啊！我们发现了的确先打印的hello,丁丁车,然后打印的hello,飞飞鱼，证明sayHi的确是异步函数。

三、await和async

await后面可以放任何表达式，不过我们更多的是放一个返回promise对象的表达式。注意await 关键字只能放到async函数里面

      let delay = word => {
        return new Promise((resolve, reject) => {
          setTimeout(() => {
            resolve(word)
          },2000);
        })
      }

先写个2s后执行的delay函数返回一个promise对象，然后再写一个异步函数来调用它

      let show = async () => {
        let res1 = await delay('I');
        console.log(res1);
      }
      show()

然后我们发现2s后的确打印了：

那么我们继续添加，尝试用优雅的写法

      let show = async () => {
        let res1 = await delay("I");
        let res2 = await delay("love");
        let res3 = await delay("YOU");
        console.log(res1 + " " + res2 + " " + res3);
      };
      show();

然后6s后，打印出了：

秒啊，用优雅的同步写法完成了异步操作。
这里运行的步骤是show函数执行，然后发现了await，它等待第一次promise对象执行完毕，然后向下执行，又发现await，又开始等待，然后重复，最后在6s之后成功完成。
